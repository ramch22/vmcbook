On-BoardingIntroduction

Overview

This document discusses the on-boarding process for VMware Cloud on AWS. It assumes that VMware Cloud services have been activated as discussed in Getting Started.

On-boarding is defined as “The process by which the initial Software Defined Data Center is provisioned”. This definition is considered the minimum amount of activity required to establish a usable SDDC for new customers. Practically speaking, most customers require what they consider to be a “production ready” environment rather than the minimal environment established by the on-boarding process. While the exact definition of “production” varies greatly between organizations, this guide will attempt to capture the most common.

The best strategy for new customers is to browse the Project Plans and become familiar with the absolute minimal requirements to complete the on-boarding process. Use the Requirements document as a reference to the items in each project plan. From there, you should create a custom project plan which fits your specific organization. This project plan should separate optional vs required tasks at a given phase of the on-boarding process, and should assign owners and deadlines for each task.

Project Plans

The following project plans may be used as part of the on-boarding process. Plans are exportable to CSV format and may be imported into a spreadsheet application.

Pre On-Boarding

The Pre On-Boarding phase of a project focuses on education and preparation. Note that steps which are marked as “required” must be completed before proceeding to the on-boarding phase.

On-Boarding

On-Boarding focuses on service activation and the initial SDDC deployment. Note that steps which are marked as “required” must be completed before the SDDC may be used.

Post On-Boarding

Once an SDDC has been deployed, the next step is to focus on all of the tasks which are required in order to use the SDDC. The required/optional status of the tasks here greatly depend on the business cases being addressed by VMware Cloud on AWS.
